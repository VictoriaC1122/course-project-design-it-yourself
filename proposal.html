一、現況說明（Situation）

本計畫旨在解決目前屋主、建築事務所與建材供應商在建材選擇過程中所面臨的各種挑戰。 透過開發一個整合式建材平台，本專案希望提升溝通效率、價格透明度， 以及設計決策的即時性。

現行困境

目前，屋主主要依賴建築事務所提供的既有建材，導致選擇受限。 若屋主希望採用非合作供應商的材料，則必須額外協調與議價， 常造成價格不明與流程冗長。此外，屋主無法直覺地感受到所選建材在實際空間中所營造的氛圍， 必須等設計公司完成3D模擬後才能確認，進而導致反覆修改與時程延誤。

對建築事務所的影響

客戶常難以想像材料在實際空間中的樣貌，導致多次設計修正、勞務成本與溝通成本增加。 設計師需投入大量時間進行後期修改，無法專注於核心創意工作。

對建材供應商的影響

合作通常需逐一拜訪設計事務所進行推廣，讓中小品牌或知名度較低的廠商難以有效進入市場。 缺乏直接接觸消費者的管道，市場機會受限。

二、問題分析（Problems）

市場上雖有多種解決方案試圖處理「建材選擇與預視」的課題，但各方案仍有侷限：

現有解決方案的局限

1. 專業室內設計服務

  • 提供客製化設計與高品質渲染圖，但成本高昂
  • 需與設計師多次討論確認，流程冗長
  • 若採用非合作廠商建材，需額外協調，增加溝通與採購成本

2. 傳統3D建模與渲染軟體（AutoCAD、3ds Max等）

  • 可產出極為真實的效果，但操作複雜、學習門檻高
  • 僅適合具建築背景的專業人士
  • 一般屋主或無設計經驗者難以使用
  • 無法直接解決建材採購問題

3. 開發商或供應商提供的參考照片與樣品

  • 可展示材質紋理，但受光線、攝影條件與場景佈置影響
  • 實際安裝後效果常與樣品差異甚大
  • 無法模擬與既有空間元素的整合
  • 需大量想像與試誤

4. 生成式AI影像模擬（如 DALL·E、Midjourney）

  • 能快速生成概念圖與氛圍模擬
  • 但AI生成畫面常出現佈局錯誤、材質貼圖不準、細節不符等問題
  • 生成影像無法與真實商品規格連結
  • 僅具參考價值，尚不具實際設計輔助功能

核心缺口

現有解決方案的共通缺點：
① 對無建築或設計背景的屋主而言，操作不友善，技術或成本門檻高。
② 多數方案仍需客戶或設計師自行與供應商協調，流程繁瑣耗時。
③ 現行選項非昂貴且耗時，即快速但缺乏精準性與供應商資料。

三、解決方案（Solution）

「讓每個人都能親手打造理想空間」

本計畫提出一套整合數位生態系統——「可視化建材互動平台」， 結合完整的建材資料庫、即時3D渲染技術與透明化價格機制， 旨在串聯建築事務所、屋主與建材供應商三方， 打造一個直覺、互動且參與式的建材選擇體驗。

平台核心設計

（1）雙模式互動系統

平台設計兩種操作模式，兼顧不同使用者需求與技術背景：

✦ 專業導入模式（Professional Import Mode）

  • 建築事務所可匯入 CAD/BIM 模型或平面圖
  • 屋主登入後即可在既有模型中自由更換地板、牆面、家具等材質
  • 系統即時更新渲染效果、價格與產品細節
  • 業主可不影響設計進度地進行實驗與比較
  • 此模式讓業主能直覺參與設計過程，同時大幅減少事務所的溝通與修改成本

✦ 自主創作模式（Independent Design Mode）

  • 適用於DIY用戶或一般消費者
  • 可自行上傳平面圖或選擇平台提供的模板
  • 透過「拖拉、倒顏料式」的直覺介面，自由搭配建材並即時預覽3D效果與預算估算
  • 降低設計門檻，實現「人人都能設計」的參與式體驗

（2）整合建材資料庫

平台的核心是一個全國性建材資料庫，包含：

  • 建材規格：紋理、尺寸、永續指標等
  • 高擬真貼圖與3D貼面：保證視覺準確度
  • 多供應商的即時價格比較：價格透明化
  • 使用者評價與口碑資料：真實反饋

透過整合各品牌資料於單一介面中，平台能消弭市場資訊落差， 讓中小型建材商也能獲得公平曝光機會。

（3）即時可視化與預算同步

  • 內建的3D引擎可即時模擬不同建材在真實空間的呈現效果
  • 支持調整光源與視角
  • 每次更換材質，系統會自動更新預算試算表
  • 使用者隨時掌握成本變化，降低臨時變更的風險
  • 減少溝通往返，提升決策效率

（4）透明化與參與式決策機制

平台促進三方互動、資訊透明與共創式決策：

  • 屋主：能「邊看邊換、邊比邊選」，自主掌控風格與預算
  • 事務所：減少重工時間與修改次數，讓專業聚焦於設計創意
  • 建材商：即時接觸潛在客戶，提高曝光與成交機率

四、實施計畫（Implementation）

開發階段與時程

階段 工作項目 預計時程
第一階段 需求分析、技術架構設計、UI/UX設計 2-3個月
第二階段 核心功能開發（3D引擎、建材資料庫、用戶系統） 4-5個月
第三階段 建材數據整合、測試與優化 2-3個月
第四階段 試用版發佈、用戶測試與迭代 2個月
第五階段 正式上線、市場推廣、用戶獲取 持續進行

技術架構

  • 前端：React/Vue + WebGL 3D引擎
  • 後端：Node.js / Python + 雲端架構（AWS/Google Cloud）
  • 資料庫：PostgreSQL + NoSQL (MongoDB)用於非結構化資料
  • 3D渲染：Three.js / Babylon.js
  • 部署：Docker + Kubernetes

人力配置

  • 產品經理：1人
  • 全棧開發：4-5人
  • UI/UX設計：2人
  • 3D美術師/建材數據管理：2人
  • QA測試：2人
  • 運營與市場：2人

五、商業模式（Business Model）

營收來源

1. 建材商上架費與成交抽成

  • 基礎上架費：月費或年費制，根據廠商規模調整
  • 成交抽成：每筆成交金額的 2-5%
  • 預期貢獻：40-50% 的總營收

2. 事務所專業版訂閱制

  • 進階渲染功能、AI建材推薦、專案管理工具
  • 月費：NT$999-2,999 （根據使用量與功能等級）
  • 預期貢獻：30-35% 的總營收

3. 建材商廣告與曝光服務

  • 精準廣告投放、品牌推廣位置、樣品展示空間
  • 預期貢獻：15-20% 的總營收

4. B2B市場數據分析報告

  • 建材選擇趨勢、消費者偏好分析、競品情報
  • 定向銷售給供應商與設計協會
  • 預期貢獻：5-10% 的總營收

定價策略

採用分層訂閱制，針對不同客群提供差異化方案：
• 個人用戶：免費基礎版，進階版 NT$199/月
• 設計事務所：NT$999-2,999/月
• 建材供應商：基礎上架費 + 成交抽成

六、預期成效（Expected Outcomes）

定量目標（2025年）

  • 用戶數：10,000+ 註冊用戶（包括屋主、設計師、廠商）
  • 建材產品數：2,000+ 件
  • 月活躍用戶：3,000+
  • 完成的項目：500+ 個設計案例
  • 營收目標：年營收 NT$5-10百萬

定性成效

對屋主的影響

  • 參與感提升：親身參與設計過程，不再被動接受結果
  • 決策效率：從數週到數天，降低80%的決策時間
  • 滿意度增加：視覺效果更符合期待，減少後期修改

對設計事務所的影響

  • 效率提升：減少50%的修改溝通時間
  • 專業聚焦：設計師可專注創意而非材質協調
  • 競爭力增強：提供更高端的客戶體驗

對建材供應商的影響

  • 市場能見度：小品牌獲得平等曝光機會
  • 銷售機會：直接接觸潛在客戶，提高轉換率
  • 市場洞察：獲得消費者偏好與市場趨勢數據

產業與社會影響

  • 推動建築與設計產業數位化轉型
  • 建立透明、高效的建材市場
  • 促進設計參與的民主化，讓更多人能自主設計

未來發展潛力

FUTURE ROADMAP
  • AI智能推薦系統：根據用戶風格、預算自動推薦建材組合
  • 永續建材評分：整合碳足跡與環境衝擊數據
  • 行動應用擴展：推出 iOS/Android 原生應用
  • 全球化擴張：逐步進入亞太地區市場
  • AR體驗：支持實時AR預覽，讓用戶在真實空間中看建材效果

結論

本平台同時滿足兩種使用者的核心需求： 想花錢卻要參與的業主，以及想省錢但願意動手的DIY使用者。

無論專業或素人，都能透過本系統輕鬆打造理想空間， 以數位化方式重新定義「建材選擇」的樂趣與效率。

本企劃有助於推動建築產業的數位轉型與互動式決策文化， 同時為建材市場創造更多元、透明、高效的生態環境。
